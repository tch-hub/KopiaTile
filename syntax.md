# CodePix 構文ルール

CodePixは、各ピクセルの色を計算して描画する**ピクセル・シェーダー**のような仕組みを持つ、Lua言語ベースの実行環境です。

## 基本的な仕組み

ユーザーが記述したコードは、内部で暗黙的に `_evaluate_pixel(x, y, width, height)` という関数の中に組み込まれて実行されます。この関数は、描画領域のすべてのピクセルに対して1回ずつ呼び出されます。

コードの最終的な目的は、**そのピクセルに塗るべき色の番号(カラーコード)を `return` する** ことです。

```lua
-- 例: すべてのピクセルを白(1)で塗る
return 1
```

## 座標系（グリッド）

CodePixは標準で **5x5** のグリッドを想定した座標系を使用しています。

- **X座標 (`x`)**: 左端が `-2`、右端が `2` （中心が `0`）
- **Y座標 (`y`)**: 下端が `-2`、上端が `2` （中心が `0`）

## 利用可能な変数

コード内では、現在評価されているピクセルの状態を示す以下の変数を利用できます。

| 変数名 | 型 | 説明 |
| :--- | :--- | :--- |
| `x` | `number` | 現在のピクセルの X座標 (-2 から 2 まで加算) |
| `y` | `number` | 現在のピクセルの Y座標 (2 から -2 まで減算) |
| `width` | `number` | グリッドの全幅 |
| `height` | `number` | グリッドの全高 |

## 戻り値（色）について

- ピクセルの色を決定するためには、数値（`number`）を `return` してください。
- `return` を省略した場合や、条件分岐で何も返さなかった（`nil` が返った）場合、デフォルトの色として `1` が採用されます。

## コード例

**例1: 中心を通る十字を描く**
```lua
if x == 0 or y == 0 then
    return 0 -- 十字は黒
else
    return 1 -- それ以外は白
end
```

**例2: X座標とY座標の値を使った市松模様**
```lua
if (x + y) % 2 == 0 then
    return 0
else
    return 1
end
```

**例3: 座標を使った単純なグラデーション風**
```lua
-- 返り値に依存しますが、座標をそのまま色番号として使うことも可能です
return math.abs(x) + math.abs(y)
```

## 注意点
- 言語は **Lua** として評価されます。関数定義やテーブル、標準ライブラリ（`math`など）が一部利用可能です。
- Luaエンジンは `wasmoon` を使用しており、無限ループなどが起こるとブラウザがブロックされる可能性があるため、実行時は注意してください。
